## Implementation Steps

### Step 1 — Validate Plugin Name

Run the pinned helper to confirm `jx-plugin` matches `^jx-[a-z0-9]+(?:-[a-z0-9]+)*$`:

```bash
python3 plugins/jx-skill/scripts/plugin-creator.py validate-name jx-plugin
```

Expected: `{"valid": true, "plugin": "jx-plugin"}`. Halt and report on any other result.

---

### Step 2 — Collision Check + Preflight

**2a — Stale staging directory preflight:**

Check for a leftover staging directory from a prior aborted run:

```bash
ls plugins/.plugin-scaffold-tmp-jx-plugin 2>/dev/null && echo "STALE_DIR_FOUND" || echo "clean"
```

If `STALE_DIR_FOUND`: remove it (`rm -rf plugins/.plugin-scaffold-tmp-jx-plugin`) before continuing. This directory is never a real artifact; it is always safe to delete.

**2b — Collision check:**

```bash
python3 plugins/jx-skill/scripts/plugin-creator.py check-collision jx-plugin .
```

Check the result and follow the collision state table below. The table is the single source of truth for how to respond:

**Collision state table:**

| State | Response |
|-------|----------|
| `plugins/jx-plugin/` exists, no marketplace entry | Remove directory (`rm -rf plugins/jx-plugin/`), re-run Step 2b |
| Marketplace entry `name: jx-plugin` exists, no directory | Remove only the `jx-plugin` entry from `.claude-plugin/marketplace.json`, re-run Step 2b |
| Both directory and marketplace entry `name: jx-plugin` exist | Plugin already scaffolded — stop; do not re-scaffold |
| Marketplace entry with a **different name** has `source: ./plugins/jx-plugin` | Stop; investigate manually — do not auto-remove another plugin's entry |
| Neither exists | Proceed to Step 3 |

**Non-collision failure modes:**

If the helper exits with a non-collision error:
- _Invalid JSON in marketplace.json_ → fix the JSON, re-run
- _`plugins` array missing_ → fix marketplace schema, re-run
- _Repo root not found_ → verify you are running from the repository root

---

### Step 3 — Scaffold Plugin Skeleton

Run the atomic scaffold helper. It stages files, moves the completed directory into place, appends the marketplace entry, verifies the result, and rolls back on failure:

```bash
python3 plugins/jx-skill/scripts/plugin-creator.py scaffold jx-plugin . \
  --description "Plugin skeleton — scaffolding, validation, and publishing tools (in development)" \
  --category productivity \
  --author Jairosoft
```

**Expected output — required keys:**

- `success` = `true`
- `plugin` = `"jx-plugin"`
- `files_created` = list with exactly 9 entries
- `marketplace_updated` = a path whose tail is `.claude-plugin/marketplace.json` (the helper emits an absolute path)

Any `success: false` result is a failure — check the `error` key and `rolled_back` list.

---

### Step 4 — Verify Scaffold

Run structural verification. This checks: all 9 required files exist, `plugin.json` has correct `name` / `description` / `author.name`, and the marketplace has exactly one entry with matching `name`, `source`, `category`, and `author.name`.

**What it does NOT check:** manifest `version`, README or ABOUT content accuracy, marketplace `description` field.

```bash
python3 plugins/jx-skill/scripts/plugin-creator.py verify jx-plugin .
```

**Expected output — required fields** (extra keys are allowed and expected):

- `valid` = `true`
- `plugin` = `"jx-plugin"`
- `errors` = `[]`

Any result with `valid != true` or a non-empty `errors` array is a failure — fix reported errors before continuing.

## Rollback

The `scaffold` helper (Step 3) attempts an atomic rollback on failure: it removes the staging directory, removes the target plugin directory if created, and restores the original `marketplace.json` if it was already written. This rollback is **best-effort** — rollback failures are silently ignored and partial states are possible.

**Manual recovery** if the system is in an inconsistent state after Step 3:

1. Remove `plugins/jx-plugin/` if it exists:
   ```bash
   rm -rf plugins/jx-plugin/
   ```
2. Remove only the `jx-plugin` entry from `.claude-plugin/marketplace.json` using a text editor or:
   ```bash
   python3 -c "
   import json, pathlib
   p = pathlib.Path('.claude-plugin/marketplace.json')
   data = json.loads(p.read_text())
   data['plugins'] = [e for e in data['plugins'] if e.get('name') != 'jx-plugin']
   p.write_text(json.dumps(data, indent=2) + '\n')
   print('done')
   "
   ```
3. Remove any stale staging directory:
   ```bash
   rm -rf plugins/.plugin-scaffold-tmp-jx-plugin/
   ```

Steps 4–5 do not modify the scaffold. If they fail, fix the specific step without re-scaffolding.

**If Step 6 (README update) was partially or fully applied before rollback is triggered**, revert `README.md` manually:
- Restore opening sentence to `six plugins spanning...`
- Remove the `jx-plugin` row from the plugins table
- Remove the `### jx-plugin` section entirely
- Remove `/plugin install jx-plugin@jodex-plugins` from the Installation section
- Remove `/plugin uninstall jx-plugin@jodex-plugins` from the Uninstall section

`git checkout -- README.md` is safe to use here if no other uncommitted README changes need to be preserved.
