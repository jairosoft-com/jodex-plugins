Wiki-only filings (new pages, index/log updates) can proceed immediately without a plan. Code changes (modifying plugin structure, scripts, configs, source files) require creating a plan first and getting user approval before implementation.

**Why:** User wants separation between knowledge capture (fast, low-risk) and code modifications (need deliberation, planning).

**How to apply:** When user says "file this to the wiki" → proceed directly. When insight implies code changes → file to wiki for reference, then create a plan before touching code.
