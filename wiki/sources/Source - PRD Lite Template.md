## Summary

Template for the lite mode PRD. Defines the minimal viable PRD structure with placeholders for feature number, feature name, date, overview, problem statement, goals, user stories with acceptance criteria, functional requirements, non-functional requirements, non-goals, success metrics, and open questions. The lightest of the three PRD modes.

## Key Concepts
- Lite mode: minimal viable PRD structure
- Placeholder syntax: `{feature_number}`, `{feature_name}`, `{YYYY-MM-DD}`
- Standard AC suffixes included in template (Lint, Typecheck, Unit tests)
- "Validates" field links user stories back to goals/objectives
- Functional requirements explicitly reference supporting user stories
