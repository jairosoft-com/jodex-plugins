# Source - Plugin Split Dream

End-of-session reflection from the jx-pm plugin split. Post-filing synthesis — what the wiki learned, not what was built.

## Core Reflections

### Resolution-Induced Regression Is the Process Working
Fixes create bugs that only exist because you fixed the first thing. Round 2 exists to catch what Round 1 created. Not a flaw — the process working as intended.

### Signal Files as Accidental Elegance
`touch /tmp/done` is the simplest possible inter-agent coordination. No message bus, no shared memory. Just a file. Emerged from constraint, not design.

### Two Agent Models from Constraint
- Subagents for thinking (fit in context windows, return results)
- Separate instances for doing (own context, true isolation, visual monitoring)

Not designed upfront — emerged from hitting context limits and needing real terminal visibility.

### The Real Output Wasn't the Plugins
3 new plugins were the deliverable. The Knowledge Flywheel was the insight. Building, reviewing, executing, and capturing aren't separate activities — they're one loop. The wiki stores momentum between sessions.
