## Summary

A complete example PRD generated in lite mode for a "One-Click Checkout" feature (Feature ID 009). Demonstrates the full structure including overview, problem statement with quantitative evidence, SMART goals, user stories with numbered acceptance criteria, functional and non-functional requirements, non-goals, success metrics, and open questions. Serves as a reference for what the prd skill produces.

## Key Concepts
- Lite mode PRD structure and completeness
- Feature ID numbering convention (009)
- User story format: As a / I want / So that
- Acceptance criteria global numbering (AC-009-01 through AC-009-15)
- Standard AC suffixes: Lint passes, Typecheck passes, Unit tests pass, E2E tests pass
- Functional requirements linked to user stories
- Non-functional requirements with categories (Performance, Security, Availability, Accessibility)
