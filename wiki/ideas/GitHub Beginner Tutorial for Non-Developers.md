# GitHub Beginner Tutorial for Non-Developers

Create a beginner-friendly tutorial teaching non-developers how to use GitHub for project collaboration — issues, PRs, reviews, project boards — without requiring CLI or git knowledge.

## Motivation

- Team members (PMs, QA analysts, stakeholders) need to interact with GitHub repos
- They don't need to write code but need to file issues, review PRs, track work
- Existing GitHub docs assume developer audience
- Reduces friction for cross-functional team collaboration

## Possible Topics

- Creating a GitHub account
- Navigating repos, branches, files (web UI only)
- Filing and managing issues
- Commenting on and reviewing pull requests
- Using GitHub Projects / project boards
- Understanding PR status, checks, merge flow
- Basic markdown for formatting issues/comments
- Notifications and watching repos
