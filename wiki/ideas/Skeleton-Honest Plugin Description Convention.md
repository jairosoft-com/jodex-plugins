# Skeleton-Honest Plugin Description Convention

When scaffolding a plugin that has no skills yet, the marketplace description should acknowledge the skeleton state rather than asserting capabilities that don't exist.

## Key Claims

- Use wording like `"Plugin skeleton — X, Y, Z (in development)"` until the first skill ships
- Aspirational wording can become the permanent description once commands are live
- Surfaced as adversarial review finding R1-F4 during jx-plugin scaffold: original description "Scaffold, validate, and publish Claude Code plugins" overstated state
